# 达芬奇密码 — 设计交互规范 (Figma Handoff)

> **目标读者**: Figma UI 设计师。本文档可直接作为设计稿的唯一依据。
> **设计基准**: 375pt × 812pt (iPhone X)，1x 图。小程序交付时按 750rpx/@2x 适配。

---

## 0. 设计总则

### 视觉锚定

线下达芬奇密码桌游的**哑面 PVC 长条形塑料牌**放在**深灰蓝色毛毡桌游垫**上。牌为有色塑料材质——**牌的底色即是颜色**，背面与正面颜色一致，正面印数字。

### 核心规则对 UI 的影响

- **牌的颜色从背面即可看出**（塑料牌本身有色）。因此猜测时无需猜颜色，仅需猜数字/Joker。
- 牌的比例为 **1:2 竖长条形**（宽:高 = 1:2，立牌效果）。

### 强制禁止

| ❌ | ✅ 取而代之 |
|----|------------|
| 高饱和彩色、蓝紫渐变、霓虹发光 | 黑白灰 + 极少量克制功能色 |
| 卡牌反光、玻璃拟态、毛玻璃 | 哑面磨砂肌理 |
| 超大圆角 (>4px)、卡通图标 | 4px 直角微圆，无装饰 |
| 正方形或横宽卡牌 | 竖长条形 1:2 宽高比（立牌） |
| 高饱和卡片容器、阴影浮起 | 元素直接放毛毡上，细线分隔 |
| 花哨字体、文字投影 | SF Mono + PingFang SC，无特效 |
| 弹跳/旋转/粒子动画 | 仅 scale/opacity 过渡，推倒动画除外 |

## 5. 牌体组件 (game-tile)

### 5.1 尺寸（1:2 竖长条形）& 动态缩放

牌严格保持 **1:2** 宽高比（竖立牌，宽:高 = 1:2）。基准尺寸:

| 变体 | 宽 | 高 | 圆角 | 牌间隙 | 用途 |
|------|:--:|:--:|:--:|:--:|------|
| 大牌 | 48px | 96px | 2px | 4px | 自己的手牌（默认） |
| 标准牌 | 40px | 80px | 2px | 4px | 对手手牌（默认） |
| 小牌 | 32px | 64px | 2px | 4px | 3+ 对手/紧凑模式 |

#### 动态缩放规则 ★

当所有玩家手牌无法在竖屏一屏内完整显示时，**等比缩小所有牌**直至无需滚动即可全部可见。缩放比例保持 1:2 不变。

```
计算方式:
  availableHeight = 812pt - 顶栏(48pt) - 操作栏(按phase) - 安全间距
  neededHeight = Σ(每位对手行高) + 分隔线 + 自己手牌行高
  scale = min(1, availableHeight / neededHeight)

  每个牌尺寸 = 基准宽 × scale, 基准高 × scale
  牌面数字字号同步缩放: fontSize × scale
```

- 最小缩放比例: 0.55（低于此值改用可滚动布局）
- 对手之间间距、分隔线高度同步缩放
- 自己的手牌始终比对手牌大一级（基准值的 1.2×）

### 5.2 立体效果 ★ 新增

每张牌具备微妙的 3D 厚度感，模拟实体 PVC 塑料牌的体量:

```
立体效果构成:
  ① 底色层: 牌面背景色
  ② 厚度层: 牌底 2px 偏移的深色填充
     - 黑牌厚度: #0D0D10
     - 白牌厚度: #D5D5CC
  ③ 表面纹理: 0.5% opacity 噪点
  ④ 微阴影: 0 1px 3px rgba(0,0,0,0.12) 仅向下投射
  ⑤ 顶面微高光: 牌顶部 1px 渐变 rgba(255,255,255,0.04)→transparent（模拟哑面塑料对光线的微弱反应）

Figma 实现:
  组内 3 层: top-highlight(1px高) + face(主体) + thickness(2px下偏移)
  整体添加 drop-shadow: y=1, blur=3, rgba(0,0,0,0.12)
```

翻开的牌（isRevealed=true）厚度层颜色不变，但数字层保持清晰。

### 5.3 暗牌背面 ★ 修改

**牌的背面颜色 = 正面颜色**（塑料牌本身有色，背面可看出）。

```
黑色牌背面 = 背景 #1A1A1E（同正面），不印数字
白色牌背面 = 背景 #EBEBE5（同正面），不印数字

Joker 背面 = 同其颜色（黑 Joker 背面黑，白 Joker 背面白）
背面同样带磨砂纹理和 0.5px 边框。
```

**暗牌在界面上的表现**：

对手手牌中，未翻开的牌显示其颜色底色 + 居中 "≤" 标记（表示手牌从小到大排列顺序）：
```
黑底暗牌: 背景 #1A1A1E, "≤" 字 #4A4A52, SF Mono 16px
白底暗牌: 背景 #EBEBE5, "≤" 字 #B0B0A8, SF Mono 16px
```

## 10. 动画交付清单

### 推倒翻牌 ★ 三段式动画

牌被翻开时播放推倒动画（1.0s），模拟麻将牌被推倒的效果。这是游戏体验核心亮点。

```
时间轴 ──────────────────────────────────────►
  0ms           300ms           650ms    1000ms
  │               │                │        │
  阶段一：晃动    阶段二：推倒      阶段三：落定
  sway           push-over        settle
```

#### 阶段一：晃动（0~300ms）

牌被猜中后轻微左右晃动，表示被触碰。

- `rotate`: 0° → ±5° → ±3° → 0°，晃动 2~3 次幅度递减
- 牌边框金色发光 + 底部阴影加深
- 实现：CSS `@keyframes` 操作 `rotate`

#### 阶段二：推倒（300~650ms）

核心——牌从竖立倒下，背面渐隐，正面数字浮现。

- `scaleY`: 1.0 → 0.06（高度急剧缩小 = "倒下"）
- `scaleX`: 1.0 → 1.18（宽度略增 = "摊开"）
- `opacity`: 1 → 0.5（背面渐隐）
- Timing：`ease-in`（先慢后快，重力感）
- 对手牌：数字从无到有浮现
- 自己牌：仅推倒效果，无数字变化（玩家已知自己牌面）

#### 阶段三：落定（650~1000ms）

- `scaleY`: 0.06 → 1.0（弹回竖立）
- `scaleX`: 1.18 → 1.0（恢复）
- `opacity`: 0.5 → 1（完全显现）
- 牌面清晰展示数字+颜色

#### 触发条件

- `.was-revealed` 类仅在 `tile.isRevealed === true` 时触发
- 摸牌时（`faceUp=true`, `isRevealed=false`）**不触发**动画
- 仅翻开（猜测导致 `isRevealed` 变化）时播放一次

| 动画 | 时长 | 缓动 | 说明 |
|------|:---:|------|------|
| 牌推倒三段式 | 1000ms | ease-in | CSS `@keyframes tile-pushover` |
| 摸牌入场 | 300ms | ease-out | — |
| ▲ 三角脉冲 | 1500ms | ease-in-out | CSS `pulse-opacity` |
| 猜测面板滑入/出 | 250ms | ease-out | — |
| 猜对提示浮层 | 200ms 淡入 | ease | Toast |
| 猜错提示浮层 | 200ms 淡入→2s→淡出 | ease | Toast |
| 回合切换 | 200ms 淡入淡出 | ease | 状态刷新 |
